# 計測台帳マップ（成長計測データの所在とスキーマ）

このプロジェクトの成長計測データ（PDCAの「計測」層）は、コーチ個人の Google Drive にある Google スプレッドシート4枚で管理する。本書はその所在（ファイルID）と各シートの列構成を、スキルが決定論的に読めるようにまとめたもの。各ドリルがどの指標を動かすかは [../drills/ドリル索引.md](../drills/ドリル索引.md) と対応する。

- **置き場所**: コーチ個人アカウント（tsutushi0628@gmail.com）の Drive フォルダ「南中野」（フォルダID: `1Cgiokrkv_0NVvRzLOZawlhNOLHMsB2sV`）
- **読み取り**: Drive コネクタ（個人アカウント接続時）で各シートIDを参照する
- **個人情報**: 実名は「選手名簿」の表示名列にのみ置き、プロジェクト側の参照・出力は選手IDで行う（実名をこのリポジトリに書かない）

## シート一覧

### 選手名簿（ID: `1YKdhUFXqctDGBssmGonXSJ2ldqXZUNkgl-4klYtbeL4`）

列: 選手ID / 表示名 / 性別 / 学年 / ポジション / 利き手 / 本人の目標 / 在籍状態 / メモ

- 選手ID（例: M01・F01）が全シート共通のキー。男女別は性別列で持つ。

### 計測項目マスタ（ID: `1RLVJkoePgSVvj9TmSQwMs2No2Z-Pi2GPIOoWxGZKLnU`）

列: 指標ID / 指標名 / 単位 / 測り方 / 頻度 / 紐づく能力カテゴリ / 良い方向 / 関連ドリル

- 指標IDが計測の共通キー。「関連ドリル」列が [ドリル索引](../drills/ドリル索引.md) と接続する。
- 良い方向（高いほど良い／低いほど良い）で、成長の向きを決定論で判定する。

### 目標（ID: `1DxPBN9Sp7hTSbQb7l1ks0jWHnO4spXl_ZWUEDin8c2E`）

列: 選手ID / 指標ID / ベースライン値 / ベースライン日 / 目標値 / 目標期日 / メモ

- 選手ID=「チーム」の行でチーム目標も持てる。目標期日は大会などに紐づける。

### 計測ログ（ID: `1rYClw7l9IkbA1XD84V6fZoufaBSW48e8ty2Zu7136ug`）

列: 日付 / 選手ID / 指標ID / 値 / メモ

- 1行=1計測の縦型。追記しやすく、成長＝最新値−ベースライン、達成度＝最新値 vs 目標値 を決定論で計算できる。

## つながり

- 計測項目マスタ.関連ドリル ↔ ドリル索引 … どの練習がどの指標を動かすか（計画→ドリル選択）
- 計測ログ ＋ 目標 → 成長と Fit&Gap … PDCA の Check / Act
- 選手名簿 → 選手個別アドバイスの土台（選手IDで計測ログ・目標を引く）
