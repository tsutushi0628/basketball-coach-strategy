### 伏字化ロジック（最終形＝合成円方式。途中で2度作り直した）
コート図では味方=青丸・守備=赤丸の「背番号＋実名」が**画像にラスタ焼き込み**されており、テキスト検索の黒塗りは効かない。OCRも薄い名前・丸に重なる名前を取りこぼす。そこで最終的に次の方式に到達した:
1. テキストレイヤの実名はPDF段で黒塗り（`search_for`）。
2. 色マーカー（青/赤の彩度の高い丸）を検出。
3. **各丸を半径3.2rで完全黒塗り → チーム色の合成円として描き直す**。味方(青)は背番号をOCR（チーム色で許容番号 {9,22,71,84,92} に厳格制約、誤読排除）して描画、守備(赤)は無番号の色ドット。**元ピクセルを一切復元しない**ため、名前が丸に重なっていても構造的に漏れない。
4. 丸から離れた凡例・タイトル内の名前はOCRで黒塗り。
5. コミット解像度（縮小後）で再OCRし、実名（途中読み含む）が0件になるまで `cleanup` で追加黒塗り。
- 実名→トークン: Kirara→選手A / Fuka→選手B / Junko→選手C / Mahoro→選手D / Ena→選手E / Saho→選手F / Akari.A→選手G / Akari.H→選手H / Shirayuri→相手校。

## 開発中に得た教訓
1. **画像内の伏字は OCR＋自動verify だけでは担保できない**。何が起きたか=コミット解像度の再OCRで0件だったのに、独立目視レビューが「Kirara」「Ma断片」を検出した（薄い文字・丸に重なる文字はOCRに見えない）。再発防止=画像の伏字は、コミット前に**独立した複数エージェントの全解像度目視レビュー**を必ず通す。反映先候補=このスキル／クロスセッション記憶。
2. **画像伏字で「元ピクセルを復元する」方式は、名前が丸に重なると漏れる**。何が起きたか=丸を黒塗り→丸の円盤だけ元画像から復元する方式だと、名前ラベルが背番号の丸に重なって焼かれているケースで、復元時に名前断片も戻ってしまった（半径をどう変えても番号と名前のピクセルが重なり分離不能）。再発防止=元を戻さず**合成再描画**（丸ごと黒塗り→色丸＋OCR番号を新規描画）にする。反映先候補=クロスセッション記憶。
3. **漏れ画像を一度commit/pushすると履歴から消すのはオーナーの領分**。何が起きたか=レビュー前にコミット→pushしたため、修正してもblobが履歴に残った。再発防止=PIIを含む画像は**コミット前**に独立レビューを通す。反映先候補=このスキル。
