## 概要

### このセッションで達成したこと
コーチが配る練習計画UIの「日」を、コーチ1人で男女2チームを同じ体育館で回す実態に合わせた**男女2列のタイムライン**に作り直した。中央に時計レール、男子＝左列・女子＝右列。同じ時刻の行で左右は別のドリルが並び、コーチが付くのは片方だけ（要監督ドリルを無監督にしない）、次の段で左右が入れ替わる（組違い＝スワップ）。自走の主ドリルは男女同時にやる「合同（together）」の実尺ブロック。練習時間は16:05開始〜17:45終了の100分を埋め切るようにし、年/月/目標の専門語を普通の日本語に直した。採用は3見せ方のうち「タイムライン型」のみ（オーナー決定）。

### なぜやったか
前セッション差し戻し（共通メニュー1列＋別枠の組違いパネルでは組違いを表現できていない・見づらい）を受けて、男女2列で「誰がいつコーチ付きか」を直接見せる必要があった。本セッション中にオーナーが実物をコーチ目線で精査し、次々に実態との乖離を指摘した（下記「開発中に得た教訓」）。その都度、場当たりでなく実態に合わせて直した。

## 実装した機能

### A. 男女2列ローテーション（中央時計スパイン）— timeline 採用
- **データ層**: `ui/rotation.mjs`（新規）が presentation day（`pd.blocks`）から `pd.rotation.rows` を導出する純モジュール。退役 `engine/src/groups.js`（`git show 0bf06ac:engine/src/groups.js`）の不変条件（要監督を無監督にしない／左右は別ドリル／両列が同一カリキュラム被覆）と検証は移植したが、**配分アルゴリズム(Pass A/B)は捨てて「逐次レイアウト」に**した（理由は教訓2の時間消失）。
  - `type='shared'` 行＝WU/主自走/CD（両列同一・全幅・実尺占有）。
  - `type='rotation'` 行＝要監督段を前半/後半の2行にスワップ展開（前半=コーチ側practice/裏は短い自走→後半で左右入替）。`coachSide` は round ごと交互。
  - 末尾に**「日の行minutes合計 == 窓(100分) を一致させ、不一致なら throw」の不変条件**を入れ、時間消失の再発を機械で封じた（`ui/rotation.test.js` で回帰）。
- **描画層**: `ui/render-shared.mjs` の `genderTwoColumn(pd, renderCell)`。`ui/pattern-timeline.mjs` が「左=男子・中央=時計（比例高さdot＋クロックレール）・右=女子」の3列スパインで、元の比例タイムラインの見た目を保ったまま2列化。全段に時間表示。

### B. 時間モデル（16:05開始・17:45終了・100分を埋める）
- **主自走ドリル（ツーメン等・コーチ不要）は男女同時の together ブロックで実尺占有**（裏埋めに溶かさない）。コーチ段（11メン・4対4）だけ swap。これで日が窓いっぱい(100分)埋まる。
- 開始は `ui/plan-data.mjs` の `DEFAULT_START_MIN` を 16:00→16:05（準備5分込み）。終了は engine config の平日 `schedule[].minutes`＝100（=窓100分・16:05〜17:45）。**config は最終的に未変更（committed値100のまま）**。

### C. WU集約（presentation のみ・engineデータ不変）
- ウォームアップを「ダイナミックストレッチ」1本の主見出し＋可動域micro動作（パワーウォーク／ランジウォーク／肩甲骨スキップ／肩回し／ヒップローテーション）を**内訳(components)に畳む**。分刻みの別ドリル6行をやめた。`plan-data.mjs` の集約処理＋3パターンの内訳表示。

### D. 専門語の平易化（コーチが読む普通の日本語）
- 「年間アーク」→「1年の流れ」、「山／ピーク」→「目標の大会（最初の目標／いちばんの目標）」、「主眼」→「今月やること」、「計測台帳」→「チェックする数字」、「原典」→「年間予定」、「集大成」→「本番/いちばん大事な大会」。`render-shared.mjs`（ラベル）＋ `engine/data/annual-plan.json`（peaksラベル・6/7月のphase/headline）＋ `plan-data.mjs`（assumptions注記）で置換。可視ジャーゴンは検出ゼロ（残るは非表示のCSSコメント1箇所＝`render-shared.mjs` の月セクションコメント）。

### E. 年の男女2行化

## 未解決の判断点・既知の注意点（番号＝優先度）

### 1.【最優先・未解決バグ】年の「いま」が男女でずれている（論理矛盾）
- **症状**: 年タブで「男子いま＝8月／女子いま＝9月」と1ヶ月ずれて表示される。オーナー指摘（原文趣旨）「なんで女子今と男子今がずれるの？ずれるわけねーだろ、別時空に生きてるのか」。**今は男女とも同じ暦月（8月）にいるのだから『いま』がずれるのは論理的にあり得ない。**
- **震源（修正箇所・file:行は ceb5e2f 時点）**:
  - `ui/plan-data.mjs` L373-374: `currentBoys: resolveMonth(annual,'男子',currentMonth).arcMonth` / `currentGirls: resolveMonth(annual,'女子',currentMonth).arcMonth`。女子は `_gender_offset`（`engine/data/annual-plan.json` の女子1ヶ月前倒しモデル）で arcMonth が+1される。→ **両方とも `currentMonth`（同じ暦月）にするのが正**。`currentBoys: currentMonth, currentGirls: currentMonth`。
  - 併せて「女子先行1ヶ月」の誤った前提を全注記から消す（オーナーが明確に否定）。残存箇所:
    - `ui/plan-data.mjs` L390 assumption「目標の大会の時期は男女で約1ヶ月ずれる（女子が先）ため『いま』の位置も1ヶ月ずれる…」→「今は男女同じ」系へ。
    - `ui/plan-data.mjs` L362-363 コメント（女子先行の記述）。
    - `ui/render-shared.mjs` L325 年セクション注記「女子は男子より約1ヶ月先行するので…」→ 削除/「今は男女同じ流れ」へ。
    - `ui/render-shared.mjs` L356 月セクション注記「…目標の大会の時期だけ男女でずれます（女子先行）。」→ 削除/「今は男女同じ予定」へ。
    - `ui/render-shared.mjs` L296 コメント。
  - `engine/data/annual-plan.json` の `_gender_offset`（女子7月始動＝1ヶ月前倒し）モデルは、`resolveMonth('女子')` 経由で年いまにしか使われていない（共通メニューは `resolveMonth('男子')` ベース・girlsGoals は girlsInput）。currentGirls を currentMonth 直指定にすれば offset は年表示に効かなくなる。`_gender_offset` を残すか撤去するかは要判断（残すなら「いまには効かせない」を明記）。
- **完了条件**: 年タブで男子いま・女子いまが**同じ月**に出る。「女子先行」の語が可視テキストから消える。男女2行は構造として残す（オーナーは「分けるんだよ男女に」と2行化自体は要求。今は2行が同内容でよい＝今は男女同じ）。

## 開発中に得た教訓

1. **「コーチの立場で実際に使えるか」を、出す前に主体が通しで確かめる**。本セッションはオーナーから「本当にコーチの立場で考えてる？できたって言ってるの？」と強く叱責された。テスト緑・hallmmarkゼロは形式整合に過ぎず、コーチ目線の業務妥当性（時間が窓を埋めるか・言葉が普通か・男女の扱いが論理的か）は主体が日→週→月→年を実画面で通し見しないと担保できない。**再発防止**: ビジュアル成果物は「形式整合(テスト/grep)」と「意味価値(コーチ目線の通し目視)」を分けて確認し、後者を通すまで『できた』と言わない。**反映先候補**: Wasurenagusa。
2. **時間消失バグ**: 自走の主ドリルをコーチ段の「裏埋め」に溶かすと、その実尺が wall-clock から消えて日が短く終わる（100分→65分）。自走でコーチ不要のドリルは男女同時にできる＝それ自体が時間を食う。**再発防止**: 「日の行合計==窓」を throw 不変条件にした（rotation.mjs）。**反映先候補**: 完了（コードに不変条件化済み）。
3. **中核UI/データ概念は、コードを書く前に1例で挙動合意を取る**。組違いの見せ方を都合4回外し（①男女別メニュー②同時刻に両方コーチ付き③1列＋別枠④同時刻に同じドリルで片方自走）、毎回実装後に差し戻された。**再発防止**: 行＝時刻・左右＝各列の具体表を先に見せて合意（本セッションで一部実践したが、年いまの論理は確認を飛ばして実装し誤った）。**反映先候補**: 既存記憶 `project_basketball_kumichigai_2col` 強化（既反映）。
4. **確かめていない前提を「データにあるから」で表示に出さない**。「女子1ヶ月先行」は annual-plan の `_gender_offset` モデルにあったが、オーナー未確認の思い込みで、年いまをずらして論理矛盾（同じ時間にいる男女のいまがずれる）を生んだ。**再発防止**: 行動原則3（測れていない項目を代理数字で埋めない）をUI表示の前提にも適用＝未確認の差分は「同じ／未確定」で出す。**反映先候補**: Wasurenagusa。
5. **オーナーが気に入った既存デザインを勝手に作り替えない**。気に入っていた比例タイムラインを2列カードに作り替えてしまい「なんで変えた」と叱責。**再発防止**: 「左右に分ける」要求は「既存デザインを保ったまま分割」と解釈する（核を制約発言で倒さない＝既存記憶 `提案の核を制約発言で逆方向に倒さない` と同根）。**反映先候補**: 既存記憶強化。
