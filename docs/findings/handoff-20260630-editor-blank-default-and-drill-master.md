## 概要

### このセッションで達成したこと（行動変容）

コーチが週ビューから任意の日（翌週を含む）の入力画面へ進めるようになった。
これまで入力できる日は表示中の週（先頭週）の練習日だけで、今日の週から翌週の予定を打ち込む導線が無かった。

男女共通トグルをONにしても、どちらか片側だけ組んだ内容が消えなくなった。
これまでは片側だけ採用して非採用側を捨てており、往復で元の男女別が壊れていた。

入力していない日と目標が、自動の叩き台ではなく空白で表示されるようになった。
叩き台はコーチが「自動で叩き台を入れる」を押したときだけ入る。

ドリルマスタの分類を全件直した。
対人の練習にマイカン（ゴール下シュートの基礎ドリル）のような得点動作が混ざる、ファンダ（基礎）に守備や対人が混ざる状態を解消した。

4機能とも本番へ反映済みで、本番出荷前にコードレビューを実走してテナント破壊級の退行2件を本番到達前に捕まえた。

### なぜやったか

外部公開（コーチ仲間への配布）に向けて、各テナントが自分の言葉と予定で計画を組める状態に近づけるため。
既定空白化はオーナー方針「入力していないのに自動の叩き台が出るのを止める」を受けた。
ドリル分類是正はオーナー指摘「対人にマイカン等がめちゃくちゃ、全マスタを見直せ」を受けた。

## 実装した機能

### 1. 週ビューから任意日（翌週含む）の入力へ遷移（`e1d5184`）

編集できる「日」画面が表示中の週（先頭週・アンカー週）の練習日しか存在しなかったのが根因。
翌週以降は週ビューに読み取り専用グリッドでしか出ず、今日の週から翌週のメニューを打ち込む導線が無かった。

- 日レベルを `data.weeks` の全週ぶんに多週化する（週ごとのグループ＋週セレクタ）。各週の練習日を編集可能な日として描画する。
- 週グリッドの実日付ヘッダをクリック可能にし、その日の入力画面へ遷移する（`jumpToDate` で日レベルへ移り当該日を表示）。
- 日切替を実日付（ISO）起点の単一可視日制御に統一する。週起点日が未設定のテナント（全日 `date:null`）では曜日キー（`data-go`）へフォールバックして切り替える（無反応の退行を根治）。
- 日レベルの週目標バーを表示中の週へ追従させる。未来週での週目標編集が先頭週を誤って上書きする欠陥を根治する。
- 単一可視日の不変条件（編集UIが前提とする「可視の日はちょうど1つ」）を全切替経路で維持する。

#### データフロー（1）

週グリッドの日付ヘッダ押下 → `jumpToDate` が対象週グループと対象日を選び日レベルへ遷移 → その日だけを可視にして入力画面を開く。
週起点日が無いテナントでは、日付キーが無いため曜日キーで同じ単一可視日制御を行う。

### 2. 男女共通トグルの無損失化（`a287a92`）

「男女共通」をONにすると片側だけ採用する実装で、非採用側（例: 女子だけ組んだ内容）が消えていた。
男子セルに見出しが入っているとそちらが採用され、女子の実内容が落ちる事故もあった。
共通OFF時には共通内容を男女両方へ無条件複製し、元の男女別を上書き破壊していた。

- 共通ON: 中身を「両方（both）」へ無損失で集約する。片側のみ→その内容を採用、両側→items を結合（name と note が一致する行だけ重複畳み）し、見出し・枠は非空優先で拾う。両側とも空のときだけ空にする。
- 元の男女別を行内に退避する。共通セルを未編集でOFFすれば元の男女別を完全復元し、共通で編集していれば最新内容を両側へ複製する（どちらの編集も失わない）。
- 退避フィールドは保存ペイロードに載せない。保存スキーマ（行の both／男子／女子）は不変。

#### データフロー（2）

共通ON: 男女別セルを読む → 退避フィールドへ元の男女別を保存 → 無損失集約で both を作る。
共通OFF: 共通セルが未編集なら退避から男女別を復元、編集済みなら共通内容を男女両方へ複製。

| `ui/toggle-both-no-loss.test.mjs` | 片側のみ採用・両側マージ・ON/OFF往復での復元を、実保存ペイロードで検証 |

### 3. 既定空白化とオプトイン自動入力（`b2f4181`）

エンジンの叩き台を既定で表示せず、コーチが上書きした日・目標だけを表示する。
叩き台は捨てず温存し、コーチが指示したときだけ編集画面へ読み込む。

- 既定空白: コーチ上書きのある日だけ表示し、無い日は空状態に倒す（`source:'empty'`）。週・月・年の目標もコーチ上書きが無ければ「未入力」と表示し、エンジン既定見出しを出さない。
- 叩き台の温存: エンジン生成を `seedDays`・`seedFocus`・`seedHeadline`・編集用 `seedPrefill` に退避する（表示しない・捨てない）。
- オプトイン自動入力: ツールバーと空状態日の「自動で叩き台を入れる」を押すと叩き台を編集画面へ読み込み、保存でコーチ上書きになる。
- 空状態UI: 簡潔な案内と導線（入力する／自動で叩き台を入れる）。実日付の無い日（週起点日未設定）は押せない導線を出さず文言だけにする。月タブ未入力も「未入力」表示で空欄にしない。
- 保存スキーマ・多週ナビ・男女共通トグル無損失・曜日フォールバックは非回帰。

#### データフロー（3）

GET 描画: エンジン叩き台を生成 → コーチ上書きのある日/目標だけ採用し、無いものは空状態（日）／「未入力」（目標）へ → 叩き台は seed 群へ退避（表示しない）。
オプトイン: 「自動で叩き台を入れる」押下 → `seedPrefill` を編集画面へ読み込み → コーチが保存すると上書きとして残る。

| `ui/render-shared.mjs` | 空状態UI（案内＋導線・実日付の無い日は押せない導線を出さない） |

### 沈黙成功の兆候（明示）

- 既定空白化により、コーチ未編集の日は空状態・未編集の目標は「未入力」で返る。これは正常な欠損表現。「保存したのに空で返る」事象が出たら、書き込み経路の認可弾きを先に疑うこと。
- 撤去した死にカテゴリは稼働コード参照ゼロを確認済み。今後ドリル原本に旧カテゴリ値を持つ行を足すと、許可集合・写像から漏れて候補生成に乗らない。原本へ行を足すときは現行の分類体系の値だけを使うこと。

## 開発中に得た教訓

各項目「何が起きたか／再発防止／反映先」。下記1〜3は wasurenagusa へ保存済み。4は既存ルールに既知。

### 1. ナビ・選択の起点キーを任意フィールドへ移すと未設定状態で導線が死ぬ

- 何が起きた: 日切替の起点を実日付（任意・null 許容）へ移したところ、週起点日が未設定の新規テナントで切替が無反応になった。
- 再発防止: 起点キーを任意フィールドへ移すときは、空時の従来キー（曜日）へのフォールバックを残し、最小状態フィクスチャ（全日 `date:null`）でも実機検証する。
- 反映先: wasurenagusa（保存済み）。

### 2. 状態統合トグルは片側採用でなく無損失マージ＋退避復元で設計する

- 何が起きた: 男女別↔共通のトグルが片側だけ採用して非採用側を捨て、往復で元の男女別を破壊していた。
- 再発防止: 状態統合トグルは無損失マージ＋元ブランチの退避復元で設計する。実保存ペイロードと ON/OFF 往復で無損失を実証する。
- 反映先: wasurenagusa（保存済み）。

### 3. 分類カテゴリの撤去はデータ0件化だけで根治完了と判定しない

- 何が起きた: ドリルを別カテゴリへ移して撤去カテゴリを0件にしただけでは、写像・重み・許可集合・初期テンプレ・テストに撤去カテゴリへの参照が残り、コーチ自由文の候補が0件になる退行を生んだ。
- 再発防止: 列挙値の撤去は、自由文・指標の写像／重み（`focus_weights`）／許可集合／プロビジョニングテンプレ／テストの全稼働参照を全リポ grep で同型全掃討する。
- 反映先: wasurenagusa（保存済み）・CLAUDE.md 行動原則10の具体例。

### 4. 兄弟プロジェクトの中核変更は本番前のコードレビューと敵対検証でテナント破壊級退行を捕まえる

- 何が起きた: コードが緑でも、新規テナント空状態の死んだボタンとコーチ自由文の候補0件という、テナントを壊す退行が残っていた。本番前のコードレビュー（effort high・対象リポ差分を明示）と敵対検証で本番到達前に捕まえた。
- 再発防止: コード緑＋単体テストだけで「正しく動く」と判定しない。中核変更は本番前にレビューと敵対検証を通す。
- 反映先: 既に CLAUDE.md の push 前レビュー・行動原則7に既知。
