> 本セッションは「コーチが配る練習計画表を、自動生成のたたき台＋コーチの特定日上書き＋そのまま1枚配布できる形に近づける」インプットから始まり、
> (1) 配布物からの無意味表示の除去、(2) コーチ上書き日の3層構造化、(3) 火(06/23)再編・水木(06/24/25)追加、(4) レベル別期間ピッカーへのIA刷新、(5) 印刷の1日1ページ化、まで進んだ。

## 概要

### このセッションで達成したこと（行動変容）
コーチが配る練習計画表が「自動生成のたたき台 × コーチの特定日上書き × そのまま1枚配布」に近づいた。配布物から無意味な表示（自動生成注記・「コーチ指定」ラベル・常時繰り返される「（男女共通）」・時刻の二度出し）が消え、画面にはレベル別（日/週/月/年）のナビゲーションと月/週の目標表示が付いた。印刷は選択中の1日だけを1ページに収める形になった。

### なぜやったか
前セッションでコーチ上書き日の第1号（火 06/23）を出したが、配布物として見ると自動生成のたたき台向けの表示が混ざっていてノイズが多かった。「コーチがそのまま配れる1枚」に仕上げるため、配布ノイズの除去・上書き日の見え方・ナビゲーション・印刷を一通り整えた。

## 実装した機能

### A. 配布ノイズ除去（全ビュー）
- ページ末フッターの「エンジン生成（自動）…次版で対応」注記を撤去。
- 「コーチ指定」表示を3系統（日ヘッダのバッジ・週ビューの SHARE_LABEL・貼付テキスト2箇所）すべて撤去。CSSコメントの「コーチ指定」も改名（不可視）。
- 「（男女共通）」の常時リピートを、日の狙いラベル・月見出し・月末注記・トップ説明文・貼付ヘッダ・目標見出しから除去（目標見出しの「チェックする数字は各チーム別」は残す）。
- コーチ上書き2列日の中央クロックの時刻二度出し（spine-half の from–to）を除去。

### B. コーチ上書き日の3層構造
- 第一層＝カテゴリ＋場所の見出し（cell.label）／第二層＝種目（item.name）／第三層＝説明（item.note）。
- 描画は既存 `twoColTimeline`、データは `overrides.json` の cell 構造で表現。

### C. 火(2026-06-23)の再編
- 16:00-16:25 屋外＝ラントレ（both・走り込み・アジリティ）
- 16:25-17:00 男子=？／女子=ファンダメンタルドリル（屋内）：ツーボールハンドリング(ボールを二つつかってドリブル練習)・ツーボールパス(二人一組で片手パス)
- 17:00-17:20 男子=？／女子=アラウンドシュート：カールからゴール下／カールから正面／レイアップ／エルボー→ゴール下スキップ（バウンズ・フロート→タップ）／キャッチ&シュート（対面0°・45°）
- 17:20-17:25 男子=？／女子=リバウンド：リバウンド→ゴール下
- 17:25-17:50 男子=オールコートゲーム（全面）／女子=ゲーム（予）
- 男子は全時間「？」（先代コーチ管轄でノータッチ）。court=「屋外→全面」、aim=「既存の練習を正しく整理して、より上手くなれるようにする」。

### D. 水(2026-06-24)・木(2026-06-25)の追加
- コーチ不在・半面・16:05-18:05。男子=？／女子=「男子に従う」。
- aim は元日の自走文を継承。

### E. コート表記パターンの統一
- 全面／半面／屋外→全面 の3種に統一。

### F. 日付の yyyy/mm/dd 化
- `plan-data.mjs` の `dateLabelMD` → `dateLabelYMD`。ヘッダ・日ピッカーに反映。

### G. 大見出しの撤去とフッター化
- 大見出し（h1「学校名 練習タイムライン」）を撤去。
- 小フッター `<p class="foot">学校名　練習タイムライン</p>` を `render()` 本体末尾へ配置。`.foot` CSS は `render-shared.mjs` の BASE_CSS。

### H. 月/週の目標バー
- 「この日の狙い」の上に [月の目標｜週の目標] の横2分割バー（`render-shared.mjs` の `goalsBar()`・値は `session.goals.monthMain` / `.week`）。日レベルのピッカー直下に1回。**印刷時は非表示**（画面専用）。

### I. IA刷新（レベル別期間ピッカー）
- 日タブ → レベル別期間ピッカーに刷新。
  - 日＝曜日ボタン（日曜始まり・日月は練習なしでグレー(pk-off)・火〜土が選択可で日付つき・`cal-go` クラスで日切替）
  - 週＝週頭リスト「yyyy/mm/dd〜」（データ週がアクティブ・他グレー・現状非対話）
  - 月＝表示月から半年「yyyy/mm」（今月アクティブ・他グレー）
  - 年＝1枚レポート（変更なし）
- 関数は `pattern-timeline.mjs` の `dayPicker` / `weekPicker` / `monthPicker`、CSS は PATTERN_CSS の `.picker` / `.pk`、曜日色トークン `--sat` / `--sun` は `render-shared.mjs` の TOKENS。
- ※月カレンダー（`monthCalendar`）は一旦実装したがオーナー判断で撤去（`--sat-soft` / `--sun-soft` は現状未使用）。

### J. 印刷/PDF
- 印刷は選択中の日のみ（`@media print` から全日/全レベルを展開する規則と `page-break-after` を撤去）。
- 1日=1ページ（`@page` margin 8mm ＋ `.wrap` zoom .92 ＋ 目標バー `display:none`）。
- `clientScript` の日切替セレクタを `.daytab` → `.cal-go` に変更。
