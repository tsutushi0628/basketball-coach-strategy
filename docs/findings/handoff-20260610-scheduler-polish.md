## 開発中に得た教訓

### 1. 設計仕様の途中更新レース

**何が起きたか**: 実装着手後に仕様正本が更新されたが、実装者への差分通知なしに進めたため、直前の差し戻しと噛み合わない完了報告が2回来た（旧指示への再報告）。

**再発防止**: 実装着手後に仕様正本を更新したら、即座に実装者へ「現行ファイル再Read＋差分明示」の通知を送る。完了報告が直前の差し戻しと噛み合わない場合はメッセージレースを疑い、メインが実測してから差し戻す。

**反映先**: wasurenagusa（id: mq9idbo1-7411）

### 2. 「調査メモ露出ゼロ」は固定語grepで閉じない

**何が起きたか**: 固定語grepが「全0件」を返した後も、実画面目視・全量アサートテストで3回追加漏れが見つかった。「確認」「妥当」は正当文誤爆のため除去語リストへの追加禁止（DEC-014「パス回しで動きを確認」・TDF-017「導入が妥当」が生存すべき文）。

**再発防止**: テキストクレンジングのゲートは「全量データへの全量アサートテスト（ユニットテスト）」に置き、誤爆防止の生存確認（除去語リストに入れてはいけない語の特定）をセットで行う。

**反映先**: wasurenagusa（id: mq9idkjj-6907）

### 3. PowerShell の [System.IO.File]::ReadAllText は絶対パス必須

**何が起きたか**: `Set-Location` が効かず、プロセスCWD基準で動くため、相対パスを渡すと読み込み失敗→`$null` grep が「全0件」の偽陰性を返した（本セッションで実発生）。

**再発防止**: `[System.IO.File]::ReadAllText` / `WriteAllText` を使う場合は必ず絶対パスを渡す。

**反映先**: wasurenagusa（id: mq9idp9u-d33f）

### 4. headless Chrome のタブ切替スクショはリスナー付き要素をclickする

**何が起きたか**: コンテンツコンテナ側の data 属性をクリックしてもタブが切り替わらず、同一バイトサイズの PNG が2枚出力された。

**再発防止**: クリックリスナーが付いたタブ実体セレクタを load 後にクリックする注入コードが必要。同一バイトサイズの PNG が出たらまずタブ切替失敗を疑う。

**反映先**: wasurenagusa（id: mq9idw8u-b7ef）
